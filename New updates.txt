POC Scope Document
Dynamic Toll + Corridor + Emission Impact Simulator
Built on TransOps + Bentley iTwin + Cesium
 
1. Objective
Develop a Proof of Concept (POC) that extends the existing TransOps operational toll dashboard into a Scenario Simulation Layer. The POC will enable dynamic testing of toll pricing, corridor capacity, emission impact, and enforcement sensitivity before real-world implementation.
2. Existing System Context
TransOps currently performs operational monitoring and KPI visualization based on clean input data. It calculates and presents metrics such as vehicle classification, anomaly rate, evasion rate, and revenue leakage. It does not include predictive or simulation capabilities.
3. Proposed Enhancement – Scenario Simulation Layer
A new 'Scenario Lab' layer will be built on top of TransOps. This layer will simulate future policy and operational changes using structured input parameters and engineering-grade modeling logic. Results will be visualized in both KPI dashboards and Cesium-based 3D digital twin.

MODULE 4 — Asset Health Monitoring (Interactive Infrastructure Intelligence Layer)
1. Objective
Extend the toll corridor digital twin from traffic monitoring to infrastructure lifecycle visibility by enabling interactive asset-level health inspection and condition assessment directly within the 3D environment.
This module transforms the corridor from a traffic simulation model into an infrastructure asset intelligence platform.
________________


2. Scope of Assets Covered (POC Level)
For the POC, include representative asset categories:
• Toll gantries
• ANPR / enforcement cameras
• Weigh-in-Motion (WIM) sensors
• Loop detectors
• Lane controllers
• Variable Message Signs (VMS)
• Weather station unit
• Pavement segment (road surface as an asset)
• Electrical cabinet / control box
Each asset must be individually clickable within Cesium.
________________


3. User Interaction Flow
User clicks on an asset in the 3D environment → Asset Health Panel opens.
The panel should include:
A. Asset Metadata
* Asset ID

* Asset Type

* Manufacturer

* Installation Date

* Expected Lifecycle (years)

* Last Inspection Date

* Warranty Status

B. Operational Metrics (Mocked or Simulated for POC)
   * Current Health Score (%)

   * Operating Temperature

   * Error Frequency (last 30 days)

   * Uptime %

   * Load Factor (vehicles processed per hour)

   * Environmental Exposure Index

________________


4. Model Calculation Logic
For POC, use a structured but simplified health scoring formula:
Health Score =
(0.35 × Age Degradation Index)
      * (0.25 × Operational Load Index)

      * (0.20 × Environmental Stress Index)

      * (0.20 × Fault/Error Index)

Where:
Age Degradation Index
= Asset Age / Expected Lifecycle
Operational Load Index
= Vehicles Processed ÷ Design Capacity
Environmental Stress Index
= Weather Intensity × Exposure Factor
Fault/Error Index
= Error Events per Defined Time Window
Normalize to 0–100 scale.
________________


5. Outputs / KPIs
• Asset Health Score (%)
• Risk Category (Green / Amber / Red)
• Remaining Useful Life (RUL Estimate)
• Maintenance Urgency Level
• Failure Probability (next 30 days)
Optional:
• Estimated downtime risk
• Revenue risk if asset fails
________________


6. Cesium Visualization Requirements
• Assets color-coded by health (Green > 75%, Amber 50–75%, Red < 50%)
• Hover tooltip shows:
         * Asset Name

         * Health %

         * Risk Level

• Click → Detailed Asset Panel
• Flash indicator for critical assets
• Optional icon overlay (wrench or warning symbol)
This module should visually demonstrate that the digital twin understands infrastructure condition, not just traffic flow.
________________


MODULE 5 — Predictive Maintenance & Failure Impact Simulation
1. Objective
Enable forward-looking simulation of asset degradation and maintenance forecasting based on selected traffic and environmental scenarios.
This module connects policy changes to infrastructure lifecycle impact.
________________


2. Scenario Triggers
User modifies:
• Toll increase
• Truck traffic increase
• Lane closure
• Weather intensity
• Operating hours
• Traffic growth percentage
The system calculates stress impact on assets.
________________


3. Model Calculation Logic
Step 1 — Compute Stress Multiplier:
Stress Multiplier =
(0.4 × Traffic Volume Index)
            * (0.3 × Vehicle Weight Factor)

            * (0.2 × Weather Severity Index)

            * (0.1 × Operating Time Factor)

Step 2 — Degradation Rate:
Adjusted Degradation =
Base Degradation Rate × Stress Multiplier
Step 3 — Updated Remaining Useful Life:
New RUL =
Current RUL – (Adjusted Degradation × Time Horizon)
________________


4. Predictive Outputs
For selected time-forward simulation (e.g., 30 days, 90 days):
• Assets predicted to cross maintenance threshold
• Failure Probability (%)
• Estimated Time to Failure
• Preventive Maintenance Recommendation
• Revenue Impact if Asset Fails
• Safety Risk Indicator
________________


5. Interactive Simulation Controls
Include:
• “Simulate 30 Days Forward”
• “Simulate 90 Days Forward”
• Adjustable Traffic Growth Slider
• Extreme Weather Toggle
________________


6. Cesium Visualization
• Time-forward animation (asset color transitions)
• Highlight predicted failure assets
• Lane shutdown animation if asset failure affects traffic
• Heatmap showing asset stress concentration
________________


Platform-Level Scenario Comparison
Important architectural note:
Scenario Comparison Panel should not be a standalone module.
It should be platform-wide.
For any module (Toll, Capacity, Asset Health, Maintenance, Emission):
User should be able to:
• Save Scenario A
• Save Scenario B
• Compare KPIs side-by-side
Comparison dimensions may include:
Revenue
Congestion
Emission
Asset Health Score
Maintenance Cost Projection
Failure Risk
This ensures consistency across the platform.
