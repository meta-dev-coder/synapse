POC Scope Document
Dynamic Toll + Corridor + Emission Impact Simulator
Built on TransOps + Bentley iTwin + Cesium
 
1. Objective
Develop a Proof of Concept (POC) that extends the existing TransOps operational toll dashboard into a Scenario Simulation Layer. The POC will enable dynamic testing of toll pricing, corridor capacity, emission impact, and enforcement sensitivity before real-world implementation.
2. Existing System Context
TransOps currently performs operational monitoring and KPI visualization based on clean input data. It calculates and presents metrics such as vehicle classification, anomaly rate, evasion rate, and revenue leakage. It does not include predictive or simulation capabilities.
3. Proposed Enhancement – Scenario Simulation Layer
A new 'Scenario Lab' layer will be built on top of TransOps. This layer will simulate future policy and operational changes using structured input parameters and engineering-grade modeling logic. Results will be visualized in both KPI dashboards and Cesium-based 3D digital twin.
Scenario 1 – Dynamic Toll Simulation
User Inputs:
- Toll rate changes by vehicle class (car/truck/bus/van)
- Time-based pricing (peak/off-peak)
- EV exemptions
- Enforcement intensity level
Model Calculations:
- Demand elasticity-based traffic shift
- Diversion probability
- Revenue recalculation
- Evasion sensitivity impact
Outputs (KPIs):
- Revenue delta (%)
- Traffic redistribution by lane
- Evasion rate projection
Cesium Visualization:
- Lane-level traffic density color change
- Revenue delta counter overlay
- Diversion heatmap
